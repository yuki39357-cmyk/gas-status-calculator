GAS ステータス計算機

================================
GAS + Git 運用メモ
================================

■ 前提
- ローカルで編集
- clasp で GAS に反映
- Git で履歴管理
- GAS の URL は変更しない（最新版デプロイを使う）

--------------------------------
■ GAS（clasp）関連
--------------------------------

▼ ローカル → GAS に反映
clasp push

※ push しただけで「エディタ上のコード」は即時更新される
※ Webアプリに反映されるかは「デプロイ方式」に依存

--------------------------------
■ Webアプリの反映ルール
--------------------------------

【最新版を使用】でデプロイしている場合
- clasp push した瞬間に Webアプリにも反映される
- URL は変わらない
- 再デプロイ不要

【バージョン固定】でデプロイしている場合
- clasp push しても Webアプリには反映されない
- 「新しいデプロイ」を作る必要あり
- URL が変わる

※ URLを変えたくないなら「最新版を使用」が前提

--------------------------------
■ Git 基本フロー（個人開発）
--------------------------------

① 状態確認
git status

② 変更を全部追加
git add .

③ コミット
git commit -m "変更内容を一言で"

--------------------------------
■ 履歴確認
--------------------------------

▼ 履歴一覧（よく使う）
git log --oneline

▼ 変更内容の差分を見る
git show <commitID>

--------------------------------
■ 過去バージョンを見る
--------------------------------

▼ 一時的に過去を見る（安全）
git checkout <commitID>

▼ 最新に戻る
git checkout master

※ checkout 中は編集しない（見るだけ）

--------------------------------
■ よくある注意点
--------------------------------

- clasp push = Git commit ではない
- Git commit = GASには反映されない
- 両方必要

--------------------------------
■ よくある不要ファイル
--------------------------------

- 謎のファイルが出たらまず確認
git status

- 不要なら削除して commit
del ファイル名
git add .
git commit -m "不要ファイル削除"

--------------------------------
■ 基本方針（忘れたらここ）
--------------------------------

- 作業の区切りで commit
- 動いた状態は必ず commit
- push 前に commit する
- URLを守りたいなら最新版デプロイ
